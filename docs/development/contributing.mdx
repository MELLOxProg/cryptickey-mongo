## Contributing to CrypticKey

Thank you for your interest in contributing to CrypticKey! We welcome contributions from the community to help make this project better.

### How to Contribute

We welcome contributions in various forms, including:

-   **Bug Reports:** If you find a bug, please open an issue on GitHub.
-   **Feature Requests:** If you have an idea for a new feature, open an issue to discuss it.
-   **Code Contributions:** If you want to contribute code, please follow the steps below.
-   **Documentation:** If you find any errors or want to improve the documentation, please submit a pull request.

### Getting Started

1.  **Fork the repository:** Click the "Fork" button on the [GitHub repository](https://github.com/MELLOxProg/cryptickey-mongo).
2.  **Clone your fork:**
    ```bash
    git clone https://github.com/your-username/cryptickey-mongo.git
    cd cryptickey-mongo
    ```
3.  **Create a new branch:**
    ```bash
    git checkout -b my-feature-branch
    ```

### Development Workflow

1.  **Set up your development environment:** Follow the [Installation Guide](https://github.com/MELLOxProg/cryptickey-mongo/blob/main/docs/installation.mdx).
2.  **Make your changes:** Implement your new feature or bug fix.
3.  **Test your changes:** Make sure your changes work as expected and don't break anything.
4.  **Commit your changes:**
    ```bash
    git add .
    git commit -m "feat: Add new feature"
    ```
5.  **Push to your fork:**
    ```bash
    git push origin my-feature-branch
    ```
6.  **Open a pull request:** Go to the original repository and open a pull request from your forked branch.

### Coding Guidelines

-   **Follow the existing code style:** We use Prettier for code formatting.
-   **Write clear and concise code:** Add comments where necessary.
-   **Keep your changes focused:** Each pull request should address a single issue or feature.
-   **Update the documentation:** If you add a new feature, please update the documentation accordingly.
